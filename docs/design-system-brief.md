# Design System Brief

This product should feel like a premium JavaScript practice environment, not a generic startup dashboard and not an overly playful course site.

## Design Intent

Keywords:

- focused
- premium
- restrained
- editorial
- code-literate
- trustworthy

The product should feel closer to:

- a serious technical reading and practice tool
- a crafted interview-prep workspace

It should feel less like:

- a template dashboard
- a flashy AI landing page
- a toy code playground

## Product Feeling

The target emotional response is:

- "I can focus here."
- "This feels high quality."
- "I understand where to go next."
- "This helps me practice, not just browse."

## Visual Principles

### 1. Hierarchy Before Decoration

- Prompt and answer state come first.
- Explanation comes second.
- Editor/output/visualizer support the learning process.
- Decorative elements should never compete with content hierarchy.

### 2. Dense But Calm

- The interface should support technical depth.
- Use compactness carefully, especially on desktop.
- Preserve breathing room around important reading areas.

### 3. Code-First Readability

- Code blocks must feel intentional and refined.
- Use consistent spacing, line-height, contrast, and framing.
- Ensure syntax areas and explanation prose feel related but distinct.

### 4. Premium Restraint

- Use color with discipline.
- Use motion sparingly and purposefully.
- Avoid trendy surfaces that reduce credibility.

## Typography Direction

- Use a distinctive but serious heading face.
- Pair it with a highly readable body face.
- Avoid overused default-feeling font stacks if a better choice is available.
- Keep code typography crisp and consistent.

Hierarchy guidance:

- display: used rarely for landing and key section headers
- heading: strong but compact
- body: readable at long-form explanation lengths
- meta: quiet labels and support text
- code: highly legible with clear punctuation rendering

## Layout Principles

### Landing

- Strong value proposition
- Quick paths into practice
- Product trust and attribution visible but not heavy-handed
- A clear “continue learning” or “start practicing” path above the fold

### Questions List

- Fast scanability
- Search and filters feel responsive and intentional
- Cards/rows should communicate progress, topic, and difficulty without becoming noisy

### Question Detail

- This is the most important screen in the product
- Favor a practice-workstation feeling
- Desktop should feel like a focused split environment
- Mobile should keep the prompt and actions immediately understandable

### Dashboard

- Should feel like a practice hub
- Should not feel like a BI dashboard
- Emphasize recommendations, momentum, and next actions over vanity metrics

## Color Guidance

- Aim for refined neutrals with one disciplined accent family
- Support semantic feedback for correct/incorrect/status states without neon overload
- Keep contrast high in reading and code zones
- Background layering should build atmosphere gently, not dominate

## Motion Guidance

- Motion should support transitions of understanding:
  - answer reveal
  - explanation expand
  - output update
  - timeline changes
- Keep transitions snappy and controlled
- Honor reduced motion

## Component Principles

- Buttons should feel precise and confident
- Tabs should feel stable and easy to parse
- Cards should not all look identical
- Filters should behave like tools, not decoration
- Focus states should look deliberate and premium

## Accessibility Principles

- Contrast and clarity over stylistic subtlety
- Keyboard users should move smoothly through the product
- Focus indication must always be visible
- Color should never be the only status cue

## What To Avoid

- generic purple/blue AI gradients
- oversized rounded cards everywhere
- excessive glassmorphism
- overly animated widgets
- fake complexity in dashboards
- visual noise around code content

## Success Criteria

The design is succeeding if:

- the question detail page feels like the center of the product
- the user always knows the next good action
- the interface feels premium without trying too hard
- the app remains pleasant on mobile
